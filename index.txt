  <h2>1. Introducción</h2>

  <section id="sec1-1">
    <h3>1.1 Propósito</h3>
    <p>
      Esta Guía de Implementación describe los pasos técnicos necesarios para instalar, configurar
      y desplegar el sistema <strong>EDI-301</strong>, incluyendo backend, base de datos y aplicación móvil.

    <p>Esta guía cubre:</p>
    <ul>
      <li>Preparación de infraestructura.</li>
      <li>Instalación del backend (Node.js/Express).</li>
      <li>Creación y configuración de SQL Server.</li>
      <li>Configuración de variables de entorno.</li>
      <li>Compilación y despliegue de la app móvil EDI-301.</li>
      <li>Implementación en fases (QA → piloto → producción).</li>
    </ul>
  </section>

  <section id="sec1-3">
    <h3>1.3 Público objetivo</h3>
    <p>Personal de TI, administradores de sistemas, desarrolladores y equipo de soporte técnico.</p>
  </section>
</section>

<section id="sec3">
  <h2>3. Preparación (Exploración)</h2>

  <section id="sec3-1">
    <h3>3.1 Requisitos de infraestructura</h3>

    <table>
      <thead>
        <tr>
          <th>Componente</th>
          <th>Requisitos</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Servidor Backend</td>
          <td>
            Windows Server o Linux, Node.js 18+, 8GB RAM, 50GB de almacenamiento,
            acceso HTTPS.
          </td>
        </tr>
        <tr>

          <td>SQL Server 2017+, acceso desde backend, backups activos.</td>
        </tr>
        <tr>
          <td>Estaciones de desarrollo</td>
          <td>
            Flutter SDK 3.x, Android Studio o Xcode, VS Code, Git.
          </td>
        </tr>
      </tbody>
    </table>
  </section>

  <section id="sec3-2">
    <h3>3.2 Requisitos de software y cuentas</h3>
    <ul>
      <li>Acceso al repositorio GitHub de backend y app.</li>
      <li>Credenciales de administrador SQL Server.</li>
      <li>Certificado SSL (para despliegues HTTPS).</li>
      <li>Usuario de servicio para backend en producción.</li>
    </ul>
  </section>

  <section id="sec3-3">
    <h3>3.3 Confirmación de ambientes</h3>
    <ul>
      <li><strong>DEV:</strong> desarrollo local.</li>
      <li><strong>QA:</strong> pruebas internas.</li>
      <li><strong>PROD:</strong> uso real por estudiantes y colaboradores.</li>
    </ul>
  </section>

<!-- 4. INSTALACIÓN Y CONFIGURACIÓN -->
<!-- =========================== -->
<section id="sec4">
  <h2>4. Instalación y configuración</h2>

  <!-- Backend -->
  <section id="sec4-1">
    <h3>4.1 Instalación del backend</h3>
    <ol>
      <li>Clonar repositorio:  
        <pre>git clone &lt;url-backend&gt;</pre>
      </li>
      <li>Instalar dependencias:
        <pre>npm install</pre>
      </li>
      <li>Configurar archivo <code>.env</code>:
        <ul>
          <li>DB_HOST, DB_USER, DB_PASSWORD</li>
          <li>JWT_SECRET</li>
          <li>STORAGE_PATH</li>
        </ul>
      </li>
      <li>Probar conexión a base de datos.</li>
      <li>Ejecutar script de creación de tablas en SQL Server.</li>
      <li>Iniciar API:
        <pre>npm start</pre>
      </li>
      <li>Configurar PM2 en producción:
        <pre>pm2 start app.js --name "edi-backend"</pre>
      </li>
    </ol>
  </section>

  <!-- Base de datos -->
  <section id="sec4-2">
    <h3>4.2 Instalación de la base de datos</h3>
    <ol>
      <li>Crear BD <strong>EDI301</strong>.</li>
      <li>Ejecutar scripts SQL.</li>
      <li>Crear usuario con permisos limitados.</li>
      <li>Configurar respaldos automáticos.</li>
    </ol>
  </section>

  <!-- App móvil -->
  <section id="sec4-3">
    <h3>4.3 Instalación de la app móvil</h3>
    <ol>
      <li>Clonar repositorio:
        <pre>git clone &lt;url-app&gt;</pre>
      </li>
      <li>Instalar dependencias:
        <pre>flutter pub get</pre>
      </li>
      <li>Configurar endpoint:
        <pre>BASE_URL=https://qa.edi.ulv/api</pre>
      </li>
      <li>Ejecutar app:
        <pre>flutter run</pre>
      </li>
    </ol>
  </section>

  <!-- Entornos -->
  <section id="sec4-4">
    <h3>4.4 Configuración de entornos</h3>
    <ul>
      <li>Crear archivos de configuración por ambiente (DEV / QA / PROD).</li>
      <li>Asegurar accesos permitidos en firewall.</li>
      <li>Usar URLs de API distintas por ambiente.</li>
    </ul>
  </section>
</section>

<!-- 5. IMPLEMENTACIÓN INICIAL -->
<!-- =========================== -->
<section id="sec5">
  <h2>5. Implementación inicial (Piloto)</h2>
  <ol>
    <li>Seleccionar 1–2 familias EDI para pruebas piloto.</li>
    <li>Desplegar APK de QA a sus dispositivos.</li>
    <li>Monitorear logs del backend.</li>
    <li>Registrar retroalimentación y problemas detectados.</li>
  </ol>
</section>

<!-- =========================== -->
<!-- 6. IMPLEMENTACIÓN COMPLETA -->
<!-- =========================== -->
<section id="sec6">
  <h2>6. Implementación completa</h2>

  <section id="sec6-1">
    <h3>6.1 Criterios para pasar a producción</h3>
    <ul>
      <li>Casos de uso principales aprobados.</li>
      <li>Defectos críticos corregidos.</li>
      <li>Aprobación del coordinador del programa.</li>
    </ul>
  </section>

  <section id="sec6-2">
    <h3>6.2 Despliegue a producción</h3>
    <ol>
      <li>Configurar servidor y BD igual a QA.</li>
      <li>Actualizar variables de entorno.</li>
      <li>Iniciar backend con PM2.</li>
      <li>Compilar app:
        <pre>flutter build apk --release</pre>
      </li>
      <li>Distribuir instalador a usuarios finales.</li>
    </ol>
  </section>

  <section id="sec6-3">
    <h3>6.3 Soporte posterior</h3>
    <ul>
      <li>Monitoreo durante 2–4 semanas.</li>
      <li>Revisiones quincenales con coordinador.</li>
      <li>Métricas: usuarios activos, errores, tiempos de respuesta.</li>
    </ul>
  </section>
</section>
